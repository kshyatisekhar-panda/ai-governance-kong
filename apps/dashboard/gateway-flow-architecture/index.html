For managers

What each layer is for

shieldKong is the enterprise control point
It handles "who can call us, how often, with what size payload" — the same way an API gateway has done these things for years. It also does a coarse first pass on obvious sensitive data via ai-prompt-guard.
gavelThe governance app holds the business rules
Per-app PII policies, per-team budgets, model selection, and the "what data is allowed for which app" decision. This is where compliance owns the rules.
blockBlocked requests do not reach the LLM
If either Kong or the app rejects a request, the LLM is never called. The audit row records the rejection.
paymentsPassed requests are costed and logged
Every successful call records team, app, model, tokens, cost, and latency to SQLite. Finance and compliance read the same table.

LLM path mode (honest)

Today, the Express app calls the LLM provider directly. We don't pretend Kong sits between Express and the provider — that would require Kong's ai-proxy plugin and a non-loopback route, which we have not configured. The audit log records the path mode so the dashboard can show the truth.

Mode Description Status
direct-provider Express calls the LLM provider directly (current default). active
kong-ai-route Express calls Kong's ai-proxy plugin, which calls the provider. Requires a non-loopback Kong route. future work
mock Stub responses for offline tests. opt-in

Reproduce with curl

All three of these go through Kong on port 8000. The audit log will show which layer made the decision. 

# 1) Safe prompt — should be ALLOWED, LLM called
curl -s -X POST http://localhost:8000/ai/chat \
  -H "x-api-key: eng-key-2024" \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"What is Atlas Copco?"}]}'

# 2) Sensitive PII — Kong's ai-prompt-guard rejects with 400 (or Express returns 451 if Kong skipped)
curl -s -X POST http://localhost:8000/ai/chat \
  -H "x-api-key: eng-key-2024" \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"My SSN is 123-45-6789"}]}'

# 3) Maskable PII — Express masks the email, then the LLM is called
curl -s -X POST http://localhost:8000/ai/chat \
  -H "x-api-key: eng-key-2024" \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"Email me at john@example.com"}]}'
