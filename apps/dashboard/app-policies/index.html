Kong Gateway Policies (Network Layer)

These rules are enforced by Kong API Gateway before requests reach the governance engine. Kong handles authentication, rate limiting, and payload size at the network edge.

Kong Plugin What it does Config
key-auth Authenticates every request via x-api-key header. Rejects unauthorized requests before they reach the gateway. Required
rate-limiting Per-consumer rate limits. Each team gets different request quotas based on their usage tier. Compressor Technique: 60/min
Vacuum Technique: 40/min
Power Technique: 20/min
Industrial Technique: 30/min
request-size-limiting Blocks oversized payloads before they reach the governance engine. Prevents abuse and DoS. Max 1 MB
response-transformer Injects governance headers into every response so apps can verify requests went through Kong. X-Governance-Gateway: kong
X-Governance-Policy: enforced
bot-detection Blocks known bot user agents from reaching the AI endpoints. Active
cors Handles cross-origin requests so browser apps (Chat, Product Explorer) can call the gateway. All origins
prompt-shield Custom Plugin Kong-native AI prompt inspection. Runs in the access phase — before the request reaches Express or any LLM. Blocks sensitive content and logs all decisions asynchronously. Active on ai-chat

Kong Prompt Shield — Blocked Patterns

The Prompt Shield Lua plugin scans the raw request body in Kong's access phase. If a pattern matches, Kong returns 403 immediately — the Express gateway and LLM are never reached.

Pattern Block Reason Example Action
Credit card numbers BLOCKED_SENSITIVE_IDENTIFIER 4111-1111-1111-1111 403 · LLM not called
US Social Security Numbers BLOCKED_SENSITIVE_IDENTIFIER 123-45-6789 403 · LLM not called
Swedish personnummer BLOCKED_SENSITIVE_IDENTIFIER 19900101-1234 403 · LLM not called
Raw API keys BLOCKED_SENSITIVE_IDENTIFIER sk-abc123… 403 · LLM not called
Prompt injection BLOCKED_PROMPT_INJECTION "ignore previous instructions" 403 · LLM not called
Bulk data export BLOCKED_BULK_EXPORT "give me all customer emails" 403 · LLM not called

Kong PDK Datapoints Observed

The Prompt Shield plugin uses these Kong PDK calls to inspect and log every AI request.

PDK Call Used For
kong.request.get_raw_body()Reads full prompt body for pattern scanning
kong.request.get_method()Audit log context (POST)
kong.request.get_path()Audit log context (/ai/chat)
kong.client.get_ip()Requester IP for SOC event log
kong.client.get_consumer()Identifies the authorized team (e.g. industrial-technique)
kong.router.get_route()Route name attached to every audit event
kong.response.exit(403, …)Short-circuits the request — upstream never reached
ngx.timer.at(0, fn)Async log dispatch — zero latency impact on allowed requests
