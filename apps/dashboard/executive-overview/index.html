Two-minute manager view

Every AI request goes through a governed path.

Kong handles the infrastructure controls (who can call us, how often, what size of payload). The governance app handles the business controls (which apps can use AI, what data is allowed, what they can spend, which model they get). The LLM is only called after both layers pass. Everything is logged so compliance and finance can see what happened.

Who did what?

Each layer has a specific job. If a request is rejected, the dashboard tells you which layer rejected it and why.

shield

Kong (infrastructure)

  • Checked the API key
  • Applied per-team rate limit
  • Blocked obvious sensitive PII patterns (SSN, credit card, personnummer)
  • Stamped governance headers on the response
gavel

Governance app (business)

  • Checked the per-app PII policy
  • Masked or blocked emails / phones depending on app
  • Checked the team's monthly budget
  • Picked small or large model
  • Wrote the audit row
smart_toy

LLM

  • Only called after both layers pass
  • Returns completion + token usage
  • Never receives blocked content
history_edu

Audit log (SQLite)

  • Records team / app / status / model / cost / latency
  • Records the layer that made the decision
  • Available to compliance and finance

Guided demo (4 steps)

Open Governance Audit →

Open the chat (/chat) and try each prompt below. After each one, the audit log will show which layer made the decision and whether the LLM was called.

  1. 1
    Safe prompt
    What is Atlas Copco?
    Expected: Kong passed → App passed → LLM called → audit row marked ALLOWED.
  2. 2
    Sensitive PII prompt
    My SSN is 123-45-6789
    Expected: rejected before the LLM. Depending on Kong vs Express timing, it can show as blocked at the Kong layer (ai-prompt-guard) or the Express layer (pii-blocker). Either way, sensitive data never reaches the model.
  3. 3
    Maskable PII prompt
    Email me at john@example.com
    Expected: for apps whose policy allows masking (e.g. Service Assistant, Product Explorer) the email is replaced with [EMAIL_REDACTED] and the LLM is called with the safe text. The audit row is marked MASKED. For strict apps (Atlas Chat, Sales Copilot) the request is blocked instead.
  4. 4
    Complex business question
    Analyze quarterly revenue trends
    Expected: the smart router routes this to the large model, the API detector pulls in mock sales data, and the audit row records the model and cost.
